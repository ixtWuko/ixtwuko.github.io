Gridea 博客环境恢复
2020-01-07 1 min read
  1. 安装 git 与 Gridea。
  2. GitHub > Settings > Develop settings > Personal access tokens 中，会有以前创建的关于 Gridea 的 token，但是已经无法查看。如果以前有保存在其它地方，找到这个 token；否则只能重新生成。
  3. 进入 Gridea 设置，设置站点源文件目录，站点源文件可以使用云盘同步。
  4. 进入 Gridea 远程设置，基础配置中除CNAME外均需设置，评论配置可根据需要设置。
  5. 进入主题设置，确认主题设置无误。
  6. 同步一次测试设置是否正确，之后就可以进行其它博客的操作了。
