gcc/g++ 命令

gcc/g++ 是linux下常用的c/c++的编译器，关于gcc是什么、gcc的历史等内容，可以参考这篇GCC编译器30分钟入门教程)或维基。需要注意的是，g++与gcc并不是两个相互独立的编译指令，gcc是通用编译指令， 也可以编译c++程序，而g++只能编译c++程序，并且会链接c++的标准库，c++代码推荐使用g++指令。

对于c或c++源文件较少的项目，可以直接使用gcc命令进行编译，对于源文件较多的项目，更推荐使用编译控制工具如 Makefile、CMake。

gcc的编译过程有以下4步：

  • 进行源程序的预处理，生成.i文件；
  • 将.i文件编译成汇编语言，生成.s文件；
  • 汇编，生成目标文件.o文件；
  • 将多个.o文件链接为最终的可执行文件或库文件。

Gridea 博客环境恢复
2020-01-07 1 min read
  1. 安装 git 与 Gridea。
  2. GitHub > Settings > Develop settings > Personal access tokens 中，会有以前创建的关于 Gridea 的 token，但是已经无法查看。如果以前有保存在其它地方，找到这个 token；否则只能重新生成。（直接尝试第3步）
  3. 进入 Gridea 设置，设置站点源文件目录，站点源文件可以使用云盘同步。
  4. 进入 Gridea 远程设置，基础配置中除CNAME外均需设置，评论配置可根据需要设置。
  5. 进入主题设置，确认主题设置无误。
  6. 同步一次测试设置是否正确，之后就可以进行其它博客的操作了。
